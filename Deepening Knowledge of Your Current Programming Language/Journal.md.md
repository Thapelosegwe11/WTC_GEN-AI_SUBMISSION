# Introduction  

In this exercise, I used AI to help me improve my understanding of JavaScript at a deeper level. Instead of just asking AI to generate answers, I used it to analyze my code, suggest improvements, and explain concepts more clearly.

The exercise included three activities:

1. Improving my code to be more idiomatic  
2. Reviewing older code for quality issues  
3. Learning an advanced JavaScript feature  

This journal summarizes what I learned from each activity and how it changed the way I think about programming.

---

# Activity 1: Idiomatic Code Transformation  

For this activity, I selected a JavaScript function I had written and asked AI to improve it to make it more idiomatic. The AI suggested improvements such as using modern ES6 features, simplifying logic, and improving readability.

## Key Learnings  

### 1. Idiomatic Code Reflects Language Style  

I learned that every programming language has a preferred way of writing code. In JavaScript, using arrow functions, array methods like `.map()` and `.filter()`, and proper variable declarations (`const` and `let`) makes the code more natural and professional.

### 2. Modern Features Reduce Complexity  

By using newer JavaScript features, my code became shorter and easier to understand. I realized I had been writing longer logic than necessary. Learning to use built-in methods effectively reduces unnecessary loops and conditionals.

### 3. Clean Code Improves Communication  

Code is not just for machines â€” it is for other developers too. When I compared my original version to the improved one, I saw how structure and clarity make a big difference. Writing idiomatic code improves teamwork and long-term maintainability.

# Activity 2: Code Quality Detective  

In this activity, I reviewed older JavaScript code that I had written a few months ago. AI helped me identify weaknesses and areas for improvement.

## Key Learnings  

### 1. I Now Notice Code Smells  

AI pointed out issues like:
- Long functions doing too many things  
- Unclear variable names  
- Repeated logic  
- Lack of proper error handling  

Before this exercise, I would not have easily noticed these problems. Now I can identify them more confidently.

### 2. Structure Is More Important Than Cleverness  

Some of my old code worked, but it was difficult to read. I learned that writing simple and clear code is better than writing complex logic that only I understand. Breaking code into smaller, focused functions improves quality.

### 3. Reviewing Code Should Be Intentional  

I created a personal checklist for future reviews:
- Is the code readable?
- Does each function have one responsibility?
- Are variable names meaningful?
- Is error handling included?
- Is there unnecessary repetition?

This activity helped me think more like a developer who evaluates quality, not just functionality.

---

# Activity 3: Understanding Async/Await  

For this activity, I focused on understanding `async/await` in JavaScript, since asynchronous programming can be confusing.

## Key Learnings  

### 1. Async/Await Makes Asynchronous Code Easier  

I learned that `async/await` is built on Promises but makes asynchronous code look more like regular step-by-step code. This improves readability and reduces nesting.

### 2. Error Handling Is Essential  

Using `try...catch` with async functions is very important. Without proper error handling, applications can fail silently or behave unpredictably. I now understand why structured error handling matters in real-world applications.

### 3. Understanding How JavaScript Handles Tasks Builds Confidence  

Learning how asynchronous operations work helped me understand how JavaScript handles background tasks. I now feel more confident working with APIs and asynchronous data.
